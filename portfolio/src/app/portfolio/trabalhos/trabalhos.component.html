Detalhes do Projeto

O projeto LISTA DE TAREFAS foi feito pensando na necessidade das pessoas de controlarem suas tarefas a fim de monitorarem seu fluxo de trabalho ou de sua equipe. Para os líderes de equipe, ao fazerem isso, eles podem verificar facilmente se há algum gargalo no processo e tomar medidas necessárias. Isso minimiza o tempo de inatividade das pessoas, ajudando a melhorar o desempenho.

Ao incluir uma nova tarefa, o usuário poderá detalhar sua DESCRIÇÃO, incluir o NOME do responsável pela tarefa e também a ETAPA que ela se encontra (a fazer, fazendo ou feito).

No momento da inserção ou edição da tarefa, o usuário poderá ter uma prévia da visualização desta tarefa, verificando que, para cada etapa, é atribuída uma cor para no card da tarefa (vermelho para tarefas 'a fazer', azul para 'fazendo' e verde para 'feito'.

Detalhes do Projeto

O projeto CADASTRO DE CLIENTES foi criado para demonstrar uma rotina muito comum no dia a dia das empresas: o cadastramento de clientes. É possível incluir no cadastro dados como: nome, email, telefone e endereço.

O projeto foi construído seguindo as boas práticas de desenvolvimento back-end de softwares, com uma arquitetura MVC composta por uma API REST em Java e Spring. Ao salvar o cadastro do cliente, as informações são persistidas no banco de dados (MySQL) que está hospedado em um servidor no ambiente Railway.

O fron-end do projeto foi construído em Angular (Typescript) seguindo as boas práticas de arquitetura de componentização, o que dá mais flexibilidade, desacoplamento, facilidade de manutanção e ampla possibilidade de evolução do projeto.

Detalhes do Projeto

O projeto CALCULADORAS foi criado para demonstrar as quatro operações básicas da matemática dispostas em dois modelos de calculadoras com layouts diferentes cujas operações serão realizadas de duas maneiras:

  • A primeira calculadora (básica) possui campos de input onde o usuário poderá digitar os números a serem considerados na operação matemática e posteriormente poderá clicar em qual operação (soma, subtração, multiplicação ou divisão) deseja realizar.
  • A segunda calculadora (calculadora de mão) refere-se a uma calculadora tradicional, onde o usuário deverá clicar nos números a serem considerados no cálculo matemático e posteriormente clicar em qual operação deseja fazer.

A interface (página) onde as duas calculadoras estão disponíveis para uso, foi desenvolvida em Angular, bem como a Calculadora Básica. Já a Calculadora de Mão (segunda calculadora) foi desenvolvida em Javascript puro.

Detalhes do Projeto

O projeto Juliana Korban trata-se de um site institucional criado para a consultadora de imagem e estilo Juliana. Com o objetivo de apresentar ao mercado seus serviços, a consultoria ajuda as pessoas a melhorarem sua aparência e expressar sua personalidade e identidade através da escolha de roupas, acessórios, maquiagem e cabelo.

Para o desenvolvimento deste projeto foi feita a criação da logomarca, bem como a paleta de cores da marca e os templates utilizados no site. O projeto foi construído a partir de um mapeamento detalhado do conceito da marca, dos serviços oferecidos, do seu posicionamento de mercado e da percepção de valor dos clientes.

A partir daí foi possível desenvolver um site que apresentasse ao públic a mensagem desejada pela consultora. Cada elemento: cores, ícones, imagens, textos e animações foram pensados e projetados a fim de que transmitissem uma mensagem de profissionalismo, confiança e credibilidade a fim de elevar o valor percebido dos serviços e alinhar tudo ao conceito proposto pela marca.

Detalhes do Projeto

O projeto Equiparts Brasil trata-se de uma loja virtual (e-commerce) criada para a marca Equiparts que possui uma loja física na cidade de Uberlândia. O objetivo deste projeto era ampliar a área de atuação em vendas da empresa, por meio de um canal de atendimento que ultrapassasse as barreiras geográficas.

Foram desenvolvidos templates personalizados para a plataforma, paleta de cores e iconografia. Desde o cadastro do cliente até o check-out de pagamento da compra cada etapa foi planejada para que o usuário tivesse a melhor experiência possível. A UI (interface do usuário) também foi pensada para proporcionar uma experiência intuitiva, leve e objetiva.

Além da plataforma de vendas, foi desenvolvido também uma área administrativa para o gerenciamento da loja com funcionalidades como: cadastro de produtos, gerenciamento de estoques, cadastro de cupons promocionais e dashboards de resultados para acompanhamento dos gestores.

Detalhes do Projeto

O projeto Paixão Biju trata-se de uma loja virtual (e-commerce) criada para a marca Paixão por Biju que possui uma loja física na cidade de Uberlândia. O objetivo deste projeto era ampliar a área de atuação em vendas da empresa, por meio de um canal de atendimento que ultrapassasse as barreiras geográficas e levasse a todas as pessoas a qualidade dos produtos e o excelente atendimento oferecido pela loja.

Foram desenvolvidos templates personalizados para a plataforma, paleta de cores e iconografia. Cada detalhe foi pensado para transmitir ao público o conceito de refinamento e de qualidade tão primados pela marca.

Desde o cadastro do cliente até o check-out de pagamento da compra cada etapa foi planejada para que o usuário tivesse a melhor experiência possível. A UI (interface do usuário) também foi pensada para proporcionar uma experiência intuitiva, leve e objetiva.

Além da plataforma de vendas, foi desenvolvido também uma área administrativa para o gerenciamento da loja com funcionalidades como: cadastro de produtos, gerenciamento de estoques, cadastro de cupons promocionais e dashboards de resultados para acompanhamento dos gestores. Além de tudo isso, foi realizado também a integração do e-commerce com a plataforma de backoffice do cliente (ERP).
